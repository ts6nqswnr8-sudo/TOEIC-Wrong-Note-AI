# 테스트 범위 및 원칙

핵심 도메인의 안정성을 극대화하고 UI 테스트 유지보수 비용을 최소화하기 위한 테스트 범위 원칙입니다.

## 1. 필수 테스트 영역 (Core Logic)
다음 영역에 대한 자동화된 유닛 및 통합 테스트는 반드시 작성합니다:
- **입력 검증 테스트**: 사용자의 오답 노트 입력값이 올바른지 검사 (예: 텍스트 길이, 올바른 보기가 주어졌는지 방어 로직)
- **오답 분류 테스트**: 문제 특성에 따른 `errorCategory` 등의 규칙 분류가 정확한지 검증
- **AI 응답 파싱 테스트**: AI로부터 전달된 비정형 데이터(JSON)가 스키마 모델에 안정적으로 매핑되는지 검증
- **저장 및 조회 로직 테스트**: DB 입출력(CRUD)이 정상적으로 이루어지는지 검증
- **예외 처리 테스트**: 외부 시스템 접속 실패, 토큰/Rate Limit 초과 시의 우회(Fallback) 및 실패 흐름 검증

## 2. 테스트 작성 기법 제한
- **외부 API는 Mock 또는 Fake를 사용합니다.** (OpenAI API 등을 테스트 환경에서 직접 의존하지 않습니다.)

## 3. 테스트 제외 영역 (UI Automation)
비용 대비 효과가 크지 않거나 변경이 잦은 영역의 자동화 테스트(E2E)는 작성하지 않습니다. UI 검증은 수동 테스트에 의존합니다:
- **Streamlit(또는 React) UI 자동화 테스트는 작성하지 않습니다.**
- **CSS, 스타일링, 레이아웃, 버튼 위치에 대한 자동화 테스트는 작성하지 않습니다.**
