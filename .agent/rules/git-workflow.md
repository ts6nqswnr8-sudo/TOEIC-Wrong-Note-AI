# Git Workflow 규칙

안정적이고 추적 가능한 소프트웨어 배포 관리를 위해 다음과 같은 Git 형상 관리 워크플로우를 사용합니다.

## 1. 브랜치 전략
- 기능 분리(Feature Branch) 방식을 사용합니다.
- 모든 새로운 기능 개발과 버그 수정은 `main` 브랜치가 아닌, 독립적인 브랜치를 생성하여 작업합니다. 
- 명명 규칙 가이드: `feature/이름`, `bugfix/이름`, `hotfix/이름`

## 2. 커밋 메시지 규칙
- 커밋 메시지는 작업의 본질을 명확히 전달하도록 작성합니다.
- 제목(첫 줄)은 동사 원형으로 시작하며 무엇이 변경되었는지 간결하게 담습니다. (예: `Add error category parsing module`)

## 3. Pull Request (PR) 및 병합 (Merge)
- 작업을 완료하면 `main` 브랜치로 PR을 생성합니다.
- (CI 구성 시) PR 생성 단계에서 린트(Lint) 및 테스트(Test) 파이프라인이 통과하는지 검증합니다.
- Merge 후 사용된 브랜치는 삭제하여 브랜치 구조를 깨끗하게 유지합니다.
