# 보안 규칙

애플리케이션과 사용자의 데이터, 핵심 인프라 자원을 보호하기 위한 필수 원칙입니다.

## 1. 민감 정보 하드코딩 금지
- **API 키를 코드에 직접 작성하지 않습니다.**
- OpenAI 등의 모든 권한 토큰과 인증 정보는 시스템 환경 변수(Environment Variables)로 주입받아야 합니다.

## 2. 저장소 보안
- **.env와 secrets 파일을 Git에 올리지 않습니다.**
- `.gitignore`에 `.env` 및 민감 정보가 포함될 수 있는 설정 파일이 등록되어 있는지 항상 검증합니다.
- API 키 유출을 방지하기 위해 로컬 개발환경은 `.env.example`과 같이 플레이스홀더를 제공하는 템플릿만 커밋합니다. 

## 3. 지적 재산권 및 저작권 보호
- **실제 토익 기출문제를 샘플 데이터로 대량 공개하지 않습니다.**
- 시드(Seed) 데이터나 오답 프리뷰 테스트 케이스를 구성할 때는, 문법 구조만 빌린 자체 창작 문제나 제한된 소수 샘플만 사용하여 외부 저작권을 침해하지 않도록 합니다.
